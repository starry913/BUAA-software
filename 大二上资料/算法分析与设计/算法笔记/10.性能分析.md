# 性能分析

## 摊还分析（Amortized Analysis）

摊还分析用于分析一系列操作的平均性能，即使某些单次操作可能很慢，但整体平均下来可能很快。

**三种主要方法：**
1. 聚集分析（Aggregate Analysis）
2. 记账法（Accounting Method）
3. 势能法（Potential Method）

---

## 1. 聚集分析（Aggregate Analysis）

### 核心思想
计算所有操作的总成本，然后除以操作次数，得到平均成本。

### 特点
- 直接计算总成本
- 简单直观
- 适用于操作类型相同的情况

### 例子1：动态数组扩容

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int *arr = NULL;
int capacity = 1;  // 初始容量
int size = 0;      // 当前元素个数

void push_back(int val)
{
    if(size >= capacity)
    {
        // 扩容：容量翻倍，复制所有元素
        int *new_arr = (int*)malloc(capacity * 2 * sizeof(int));
        memcpy(new_arr, arr, capacity * sizeof(int));
        free(arr);
        arr = new_arr;
        capacity *= 2;
        // 扩容成本：O(capacity)，即O(n)
    }
    arr[size++] = val;  // 普通插入：O(1)
}

// 聚集分析：
// 假设插入n个元素
// 扩容发生在：1, 2, 4, 8, 16, ... 位置
// 扩容成本：1 + 2 + 4 + 8 + ... + n/2 = 2n - 1
// 总成本：n次插入 + 扩容成本 = n + (2n-1) = 3n - 1
// 平均每次插入：O(1)
```

### 例子2：二进制计数器递增

```c
#include <stdio.h>

int counter = 0;
int flip_count = 0;  // 记录总翻转次数

void increment()
{
    int temp = counter;
    int flips = 0;
    
    // 模拟二进制递增：翻转最低位的连续1
    while(temp & 1)
    {
        temp >>= 1;
        flips++;
    }
    flips++;  // 翻转第一个0
    
    counter++;
    flip_count += flips;
}

// 聚集分析：
// 从0到n-1，总共n次递增
// 第1位（最低位）：每次递增都翻转，共n次
// 第2位：每2次翻转1次，共n/2次
// 第3位：每4次翻转1次，共n/4次
// ...
// 总翻转次数：n + n/2 + n/4 + ... = 2n
// 平均每次递增：O(1)
```

### 例子3：栈的多重弹出（Multipop）

```c
#include <stdio.h>

int stack[10005];
int top = -1;

void push(int val)
{
    stack[++top] = val;  // O(1)
}

int pop()
{
    if(top < 0) return -1;  // 栈空
    return stack[top--];    // O(1)
}

// 多重弹出：弹出k个元素（或直到栈空）
void multipop(int k)
{
    while(top >= 0 && k > 0)
    {
        pop();
        k--;
    }
    // 最坏情况：O(min(k, top))
}

// 聚集分析：
// 假设有n个操作：push、pop、multipop
// 关键观察：每个元素最多被push一次，最多被pop一次
// 总pop次数（包括multipop中的）≤ n
// 总push次数 ≤ n
// 总成本 ≤ 2n
// 平均每次操作：O(1)
```

## 2. 记账法（Accounting Method）

### 核心思想
为每个操作分配一个"摊还成本"，通过"信用/存款"概念来解释为什么某些操作可以比实际成本更便宜。

### 基本概念
- **摊还成本**：为每个操作设定的"价格"
- **实际成本**：操作的真实时间成本
- **信用/存款**：摊还成本与实际成本的差值
  - 如果摊还成本 > 实际成本：产生信用（存款）
  - 如果摊还成本 < 实际成本：消耗信用（取款）

### 记账法的步骤
1. 为每个操作分配摊还成本
2. 证明：任何时刻，总信用 ≥ 0（不能透支）
3. 结论：n个操作的总摊还成本 ≥ 总实际成本

### 例子1：动态数组扩容

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int *arr = NULL;
int capacity = 1;
int size = 0;

void push_back(int val)
{
    if(size >= capacity)
    {
        // 扩容：实际成本 O(capacity)
        int *new_arr = (int*)malloc(capacity * 2 * sizeof(int));
        memcpy(new_arr, arr, capacity * sizeof(int));
        free(arr);
        arr = new_arr;
        capacity *= 2;
    }
    arr[size++] = val;  // 普通插入：实际成本 O(1)
}

// 记账法分析：
// 为每次插入分配摊还成本 = 3
// 
// 情况1：普通插入（不需要扩容）
//   实际成本 = 1
//   摊还成本 = 3
//   产生的信用 = 3 - 1 = 2（存入）
//
// 情况2：扩容插入（需要扩容）
//   实际成本 = capacity（复制旧元素）+ 1（插入新元素）
//   摊还成本 = 3
//   需要的信用 = capacity + 1 - 3
//
// 关键：在扩容前，我们已经插入了 capacity/2 个元素
// 每个元素产生了 2 个信用，总共积累了 capacity 个信用
// 这些信用正好够支付扩容成本！
//
// 证明：总信用始终 ≥ 0
// 平均每次操作：O(1)
```

### 例子2：二进制计数器递增

```c
#include <stdio.h>

int counter = 0;

void increment()
{
    int i = 0;
    // 翻转最低位的连续1，然后翻转第一个0
    while(counter & (1 << i))
    {
        counter ^= (1 << i);  // 翻转第i位
        i++;
    }
    counter ^= (1 << i);  // 翻转第i位（第一个0）
    // 实际成本 = 翻转的位数
}

// 记账法分析：
// 为每次递增分配摊还成本 = 2
//
// 每次翻转一个位时：
//   实际成本 = 1（翻转操作）
//   摊还成本 = 2
//   产生的信用 = 1（存入该位）
//
// 关键观察：
// - 每次翻转0→1时，消耗1个信用（该位之前积累的）
// - 每次翻转1→0时，产生1个信用（存入该位）
// - 由于每个位从0→1只有一次，但可能多次1→0
// - 总信用始终 ≥ 0
//
// 平均每次递增：O(1)
```

### 例子3：栈的多重弹出（Multipop）

```c
#include <stdio.h>

int stack[10005];
int top = -1;

void push(int val)
{
    stack[++top] = val;
    // 实际成本 = 1
    // 摊还成本 = 2（多付1个信用）
    // 产生的信用 = 1（存入栈中）
}

int pop()
{
    if(top < 0) return -1;
    int val = stack[top--];
    // 实际成本 = 1
    // 摊还成本 = 0（使用之前存入的信用）
    // 消耗的信用 = 1
    return val;
}

void multipop(int k)
{
    while(top >= 0 && k > 0)
    {
        pop();  // 每个pop消耗1个信用
        k--;
    }
    // 实际成本 = min(k, top+1)
    // 摊还成本 = 0（全部用信用支付）
}

// 记账法分析：
// push的摊还成本 = 2（实际1 + 信用1）
// pop的摊还成本 = 0（用信用支付）
// multipop的摊还成本 = 0（用信用支付）
//
// 关键：每个元素push时存入1个信用
//       每个元素pop时消耗1个信用
//       总信用 = 栈中元素个数 ≥ 0
//
// 平均每次操作：O(1)
```

## 3. 势能法（Potential Method）

### 核心思想
将数据结构的状态看作一个"势能函数"，通过势能的变化来分析摊还成本。

### 基本概念
- **势能函数 Φ(D)**：表示数据结构D的"能量"
- **摊还成本** = 实际成本 + 势能变化
  - 摊还成本 = 实际成本 + (Φ(D') - Φ(D))
  - 其中D是操作前的状态，D'是操作后的状态

### 势能法的步骤
1. 定义势能函数Φ(D)，满足：
   - Φ(D₀) = 0（初始势能为0）
   - Φ(D) ≥ 0（势能始终非负）
2. 计算每次操作的摊还成本
3. 证明总摊还成本是总实际成本的上界

### 例子1：动态数组扩容

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int *arr = NULL;
int capacity = 1;
int size = 0;

void push_back(int val)
{
    if(size >= capacity)
    {
        // 扩容：实际成本 = capacity（复制旧元素）
        int *new_arr = (int*)malloc(capacity * 2 * sizeof(int));
        memcpy(new_arr, arr, capacity * sizeof(int));
        free(arr);
        arr = new_arr;
        capacity *= 2;
    }
    arr[size++] = val;  // 普通插入：实际成本 = 1
}

// 势能法分析：
// 定义势能函数：Φ(D) = 2 * size - capacity
// 
// 情况1：普通插入（不需要扩容）
//   实际成本 = 1
//   势能变化 = Φ(D') - Φ(D) = (2*(size+1) - capacity) - (2*size - capacity) = 2
//   摊还成本 = 1 + 2 = 3
//
// 情况2：扩容插入（需要扩容）
//   扩容前：size = capacity/2, capacity = capacity
//   扩容后：size = capacity/2 + 1, capacity = 2*capacity
//   实际成本 = capacity/2（复制）+ 1（插入）= capacity/2 + 1
//   势能变化 = (2*(capacity/2+1) - 2*capacity) - (2*(capacity/2) - capacity)
//           = (capacity + 2 - 2*capacity) - (capacity - capacity)
//           = -capacity + 2
//   摊还成本 = (capacity/2 + 1) + (-capacity + 2) = 3 - capacity/2
//   当capacity ≥ 2时，摊还成本 ≤ 3
//
// 平均每次操作：O(1)
```

### 例子2：栈的多重弹出（Multipop）

```c
#include <stdio.h>

int stack[10005];
int top = -1;

void push(int val)
{
    stack[++top] = val;
    // 实际成本 = 1
}

int pop()
{
    if(top < 0) return -1;
    return stack[top--];
    // 实际成本 = 1
}

void multipop(int k)
{
    while(top >= 0 && k > 0)
    {
        pop();
        k--;
    }
    // 实际成本 = min(k, top+1)
}

// 势能法分析：
// 定义势能函数：Φ(D) = top + 1（栈中元素个数）
//
// push操作：
//   实际成本 = 1
//   势能变化 = (top+2) - (top+1) = 1
//   摊还成本 = 1 + 1 = 2
//
// pop操作：
//   实际成本 = 1
//   势能变化 = top - (top+1) = -1
//   摊还成本 = 1 + (-1) = 0
//
// multipop操作（弹出k个）：
//   实际成本 = min(k, top+1)
//   势能变化 = (top+1-min(k,top+1)) - (top+1) = -min(k, top+1)
//   摊还成本 = min(k, top+1) + (-min(k, top+1)) = 0
//
// 平均每次操作：O(1)
```

### 例子3：二进制计数器递增

```c
#include <stdio.h>

int counter = 0;

void increment()
{
    int i = 0;
    while(counter & (1 << i))
    {
        counter ^= (1 << i);  // 翻转第i位：1→0
        i++;
    }
    counter ^= (1 << i);  // 翻转第i位：0→1
    // 实际成本 = 翻转的位数
}

// 势能法分析：
// 定义势能函数：Φ(D) = counter中1的个数
//
// increment操作：
//   假设翻转了t位（t-1个1→0，1个0→1）
//   实际成本 = t
//   势能变化 = (原来的1的个数 - (t-1) + 1) - 原来的1的个数 = 2 - t
//   摊还成本 = t + (2 - t) = 2
//
// 平均每次递增：O(1)
```

## 三种方法对比

| 方法 | 核心思想 | 优势 | 适用场景 |
|------|---------|------|---------|
| **聚集分析** | 计算总成本再平均 | 直观简单，易于理解 | 操作类型相同或相似 |
| **记账法** | 为操作分配摊还成本，用信用系统 | 灵活，可以为不同操作分配不同成本 | 操作类型不同，需要区分 |
| **势能法** | 用势能函数表示状态能量 | 数学严谨，适合复杂分析 | 需要精确分析势能变化 |

---

## 总结

### 聚集分析的意义
- 揭示真实性能：证明虽然某些操作很慢，但平均很快
- 指导设计决策：支持使用这些数据结构
- 提供理论保证：严格的平均复杂度证明

### 记账法的意义
- 直观理解：用"存款/取款"解释性能
- 灵活分配：可以为不同操作分配不同摊还成本
- 严格证明：通过信用系统保证不"透支"

### 势能法的意义
- 数学严谨：用函数形式化表示
- 精确分析：可以精确计算每次操作的摊还成本
- 通用性强：适用于各种复杂的数据结构

### 共同思想
将昂贵操作的成本分摊到所有操作上，证明平均成本很低，从而支持使用这些高效的数据结构。
