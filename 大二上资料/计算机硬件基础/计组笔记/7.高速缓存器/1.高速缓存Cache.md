# 高速缓存Cache
## 1.Cache的原理
### 1.1程序访问的局部性原理

#### 局部性原理定义

| 概念 | 定义 |
|------|------|
| 局部性原理 (Principle of Locality) | 大量典型程序的运行情况分析结果表明，无论是存取指令或存取数据所访问的存储单元都趋于聚集在一个较小的连续存储区域中 |

#### 局部性类型

| 类型 | 英文名称 | 定义 | 典型例子 |
|------|---------|------|----------|
| 空间局部性 | Spatial Locality | 刚被访问过的存储单元的临近单位可能不久被访问 | 数组访问、指令访问 |
| 时间局部性 | Temporal Locality | 刚被访问过的存储单元可能不久又将被访问 | 临时变量访问、循环访问数组元素/变量/指令 |

#### 局部性产生的原因

| 对象 | 原因说明 |
|------|---------|
| 指令 | 指令按序存放，地址连续，循环程序段或子程序段重复执行 |
| 数据 | 连续存放，数组元素重复、按序访问 |

#### 全相联映射的优缺点

| 类型 | 特点 | 说明 |
|------|------|------|
| **优点** | 使用Cache的灵活性最大 | - |
| | Cache空闲时，新块可直接写入 | - |
| | Cache满时，可方便地选择Cache块进行替换 | - |
| **缺点** | 比较操作复杂 | 进行Cache读写操作时，需要将主存地址的块地址与Cache中**所有**Tag进行比较，以判断是否命中 |
| | 实现成本高、复杂度高 | 实现这种比较操作的电路过多、过于复杂，实现成本过高，一般无法采用 |
| | 适用范围有限 | 因此，这种方法只在Cache容量很小时才被采用 |

#### 直接映射的优缺点

| 类型 | 特点 | 说明 |
|------|------|------|
| **优点** | 实现简单 | 只需利用主存地址中的区地址，与块地址对应的Cache块中Tag进行1次比较，即可确定是否命中 |
| **缺点** | 映射关系不灵活 | 因每个主存块只能固定地对应某个确定的Cache块，会出现Cache有很多空闲，但新块不能直接写入而需要替换的现象，Cache空间的利用不充分 |

#### 组相联映射的特点

| 特点 | 说明 |
|------|------|
| 折衷方案 | 组相联映射是直接映射和全相联映射的折衷 |
| 组间映射关系 | 主存组I与Cache组I之间是一一对应关系，好似直接映射方式，简化了实现 |
| 组内映射关系 | 主存组I内的块与对应组中Cache块是任意对应关系，好似全相联映射方式，体现了一定的灵活性 |
| 优势 | 组相联映射兼顾了实现成本和灵活性 |
| 应用 | 目前主流的商业系统中采用组相连高速缓存的方式居多 |

## 3.Cache的替换策略
### 3.1缺失处理
CPU访问Cache缺失时，CPU必须等待数据装入Cache后才能访问Cache，这期间的时间损失称为缺失损失。
取出块的时间：第一个字的延迟时间（存储器访问）+ 块的剩余部分的传送时间。
Cache的存储组织对缺失损失具有很大的影响。

### 3.2替换策略

#### Cache块的替换

| 映射方式 | 访问缺失时 | 数据块可进入的位置 | 替换策略 |
|---------|-----------|-----------------|---------|
| **直接映射Cache** | 访问缺失时 | 被请求数据所在的块只能进入Cache的一个位置 | 占用该位置的数据块必须被替换掉 |
| **组相联Cache** | 访问缺失时 | 被请求数据所在块可以进入Cache某一组的任何位置 | 因此应在Cache对应组内选择一个数据块进行替换 |
| **全相联Cache** | 访问缺失时 | 被请求数据所在块可以进入Cache的任何位置 | 因此应在Cache中选择一个数据块进行替换 |

#### 替换策略算法

| 策略名称 | 英文缩写 | 英文全称 | 说明 |
|---------|---------|---------|------|
| 最低使用频率法 | LFU | Least-Frequently Used | 记录每个数据块的使用频率，替换使用次数最少的数据块 |
| 最近最少使用法 | LRU | Least-Recently Used | 记录每个数据块的相对使用情况，替换最近没有使用的数据块，与LFU相比，更注重最近没有使用的数据块 |
| 先进先出法 | FIFO | First-In-First-Out | 替换最先装入Cache的数据块 |
| 最佳置换算法 | OPT | OPTimal selection | 替换Cache中以后不再需要的数据块，如果没有这样的数据块，则选择以后最长时间内不再访问的数据块 |
| 随机法 | RAND | Random | 随机选择一个数据块进行替换 |
