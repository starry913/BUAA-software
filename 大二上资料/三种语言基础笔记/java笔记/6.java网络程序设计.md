## 一、网络编程在干嘛（必考送分）

### 1️⃣ 什么是网络编程

* **网络编程**：
  👉 使用网络通信协议，实现**不同计算机上的程序之间通信**
* 本质：
  👉 **网络进程之间的通信**

常见表述（选择题会用）：

* ✔ 利用 TCP / UDP 等协议
* ✔ 基于 Socket 的通信
* ❌ 与单机程序无关（错）

---

## 二、网络工作模式（超爱考）

### 2️⃣ C/S 与 B/S 模式

#### C/S（Client / Server）

* 客户端 + 服务器
* 典型例子：

  * Socket 程序
  * 数据库客户端/服务器
* 特点：

  * 客户端需要安装程序
  * 功能强

#### B/S（Browser / Server）

* 浏览器 + 服务器
* 典型例子：

  * Web 网站
* 特点：

  * 不用装客户端
  * 功能相对简单

👉 **选择题口诀**

* Socket → **C/S**
* 浏览器访问网页 → **B/S**

---

## 三、协议 & TCP/IP（重点）

### 3️⃣ 什么是网络通信协议

* **协议**：通信双方必须遵守的规则
  （速率、格式、顺序、出错控制等）

### 4️⃣ TCP/IP 协议体系（必背）

从低到高 **4 层**：

1. 物理层
2. 网络层（IP）
3. 传输层（TCP / UDP）
4. 应用层（HTTP / FTP / SMTP）

⚠️ 选择题常考：

* TCP、UDP 属于 👉 **传输层**
* IP 属于 👉 **网络层**
* HTTP 属于 👉 **应用层**

---

## 四、TCP vs UDP（超级重点，必考对比）

### 5️⃣ TCP 特点

* **面向连接**
* **可靠**
* **按顺序**
* 以 **字节流（stream）** 方式传输
* 常用于：

  * 聊天
  * 文件传输

### 6️⃣ UDP 特点

* **无连接**
* **不可靠**
* **不保证顺序**
* 发送的是 **数据报**
* 优点：

  * 快
  * 占资源少

👉 选择题秒杀口诀：

| 对比   | TCP | UDP |
| ---- | --- | --- |
| 是否连接 | 有   | 无   |
| 是否可靠 | 是   | 否   |
| 顺序保证 | 有   | 无   |
| 速度   | 慢   | 快   |

---

## 五、IP & 端口（几乎必考）

### 7️⃣ IP 地址

* 用来**唯一标识一台计算机**
* IPv4：32 位
  例子：`192.168.0.1`
* 本地回环：

  * `127.0.0.1`
  * `localhost`

### 8️⃣ 端口（Port）

* 用来**标识同一台计算机上的不同进程**
* 范围：`0 ~ 65535`
* `0 ~ 1023`：

  * 知名端口
  * 一般不用

常见端口：

* HTTP → 80
* FTP → 21

👉 IP + Port = **唯一的网络通信进程**

## 六、URL 与 URI（必背结构）

### 9️⃣ URL 是什么

* **统一资源定位器**（Uniform Resource Locator）
* 用来**定位网络资源**
* 不仅标识资源，还提供**访问资源的方式**（协议、主机、端口等）

### 1️⃣0️⃣ URL 五个组成部分

```
协议://主机名:端口/资源名#锚点
```

例如：

```
http://www.sina.com.cn:8080/index.html
```

对应：

* 协议：http
* 主机名：[www.sina.com.cn](http://www.sina.com.cn)
* 端口：8080（可省）
* 资源名：index.html
* 锚点：#xxx（可省）

### 1️⃣1️⃣ URI 是什么

* **统一资源标识符**（Uniform Resource Identifier）
* 用来**标识资源**
* 是一个**更广泛的概念**，包括 URL 和 URN

**URI 的组成**：

```
[协议:]//[主机名[:端口]][/路径][?查询参数][#锚点]
```

例如：

```
http://www.example.com:8080/path/to/resource?id=123#section
file:///C:/Users/file.txt
urn:isbn:978-0-123456-78-9
```

**URI 的特点**：

* URI 可以是**绝对路径**或**相对路径**
* URI 可以包含**查询参数**（?key=value）
* URI 是**标识符**，不一定能直接访问资源

### 1️⃣2️⃣ URL 与 URI 对比（必考）

| 对比点 | URL | URI |
| ------ | --- | --- |
| **全称** | Uniform Resource Locator（统一资源定位器） | Uniform Resource Identifier（统一资源标识符） |
| **作用** | 定位并访问资源 | 标识资源 |
| **关系** | URI 的**子集** | **父概念**，包含 URL 和 URN |
| **是否包含访问方式** | ✅ 是（协议、主机、端口） | ❌ 不一定 |
| **是否可直接访问** | ✅ 是（提供完整访问路径） | ❌ 不一定（URN 不能直接访问） |
| **例子** | `http://www.example.com/index.html` | `http://www.example.com/index.html`（URL 也是 URI）<br>`urn:isbn:978-0-123456-78-9`（URN，也是 URI） |

**详细解释**：

* **关系**：
  * **URI 是父概念**，URL 是 URI 的一种
  * 所有 URL 都是 URI，但不是所有 URI 都是 URL
  * URI = URL + URN（统一资源名称）
* **URL 的特点**：
  * 必须包含**访问方式**（协议、主机、端口）
  * 可以直接用来**访问资源**
  * 例如：`http://www.example.com/index.html`
* **URI 的特点**：
  * 只需要**标识资源**，不一定提供访问方式
  * 可以是 URL，也可以是 URN（如：`urn:isbn:978-0-123456-78-9`）
  * 可以是绝对路径或相对路径

**记忆技巧**：

> **URL 是 URI 的一种，URL 能定位，URI 能标识**
> 
> **所有 URL 都是 URI，但 URI 不一定是 URL**

**选择题常考**：

* ✅ URL 是 URI 的子集
* ✅ 所有 URL 都是 URI
* ❌ 所有 URI 都是 URL（错误，URI 还包括 URN）
* ✅ URL 必须包含访问方式（协议、主机等）

## 七、Socket（老师最爱抽象概念）

### 1️⃣3️⃣ Socket 是什么

* Socket 是：
  👉 **网络编程的接口**
* 是应用程序和网络之间的“接口”

老师经典比喻（会原样考）：

> Socket 就像 **港口码头**

* 程序把数据放到 Socket
* 网络负责传输
* 程序只关心“放”和“取”

## 八、Java 网络编程核心类（选择题）

### 1️⃣4️⃣ TCP 相关类

* `Socket`
  👉 客户端 + 服务器通信
* `ServerSocket`
  👉 **只用于服务器端**

✔ 基于 **TCP**

### 1️⃣5️⃣ UDP 相关类

* `DatagramSocket`
* `DatagramPacket`

✔ 基于 **UDP**

---

## 九、TCP 编程流程（概念题）

### 1️⃣6️⃣ TCP 服务器流程

1. 创建 `ServerSocket`
2. `accept()` 等待连接
3. 返回一个 `Socket`
4. 通过流通信
5. 关闭资源

### 1️⃣7️⃣ TCP 客户端流程

1. 创建 `Socket`
2. 连接服务器
3. 通过流通信
4. 关闭

👉 关键词：

* **accept**
* **一对一**
* **服务器先启动**

---

## 十、UDP 编程特点（必考一句话）

### 1️⃣8️⃣ UDP 编程特点

* 没有服务器/客户端之分
* 两端地位**平等**
* 接收程序必须先启动
* 通过 `send()` / `receive()`
