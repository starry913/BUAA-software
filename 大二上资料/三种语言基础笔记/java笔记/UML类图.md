# UML类图

UML（统一建模语言）是一种用于可视化、规范化和构建软件系统文档的标准建模语言。

## 1. 类图基本符号
- **类名**：写在矩形框的顶部
- **属性**：写在中间部分，格式为 `可见性 属性名:数据类型`
- **方法**：写在底部，格式为 `可见性 方法名(参数):返回类型`
- **可见性符号**：
  - `+` 表示 public（公共）
  - `-` 表示 private（私有）
  - `#` 表示 protected（受保护）

## 3. UML类图说明

### GirlFriend类特点：
- 所有属性都是私有的（-），体现了封装性
- 通过公共的getter和setter方法访问属性
- setAge方法包含数据验证逻辑

### Phone类特点：
- 属性是公共的（+），可以直接访问
- 包含两个行为方法：call()和playGame()

### 测试类特点：
- 与对应的类存在"使用"关系
- 负责创建对象并调用方法进行测试

## 4. UML关系类型

### 4.1 依赖关系（Dependency）
- 表示一个类使用另一个类
- 用虚线箭头表示：`---->`
- 例如：测试类依赖被测试的类

### 4.2 关联关系（Association）
- 表示类之间的结构关系
- 用实线表示：`----`
- 可以标注角色名和多重性

### 4.3 聚合关系（Aggregation）
- 表示"整体-部分"关系
- 用空心菱形表示：`<>----`
- 部分可以独立于整体存在

### 4.4 组合关系（Composition）
- 表示强"整体-部分"关系
- 用实心菱形表示：`<>----`
- 部分不能独立于整体存在

### 4.5 继承关系（Inheritance）
- 表示"is-a"关系
- 用空心三角形表示：`△----`
- 子类继承父类的属性和方法

### 4.6 实现关系（Implementation）
- 表示类实现接口
- 用空心三角形虚线表示：`△----`
- 类必须实现接口中定义的所有方法
